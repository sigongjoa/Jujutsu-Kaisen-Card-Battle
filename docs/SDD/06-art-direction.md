## 1. 아트 바이블의 목적

이 문서는 모든 아티스트(내부 직원 및 외부 프리랜서)가 준수해야 할 **시각적 헌법**입니다.

### 1.1 원칙

✅ **일관성**: 모든 카드의 스타일이 하나로 느껴짐
✅ **인식성**: 세력과 카드 타입이 시각적으로 구별됨
✅ **직관성**: 카드 정보가 명확하게 읽힘
✅ **몰입감**: 주술회전 세계관에 충실함

## 2. 전체 비주얼 톤 (Overall Tone & Mood)

### 2.1 무드 보드 (Mood Board)

**스타일**: 어둡고 세련된 다크 판타지
**색상팔레트**: 검은색, 자주색, 빨강, 금색의 강렬한 대비
**분위기**: 위협적이지만 아름다운 주술의 세계

### 2.2 참고 영상/이미지

```
무드 보드 참고:
- 주술회전 애니메이션 (Opening, ED)
- 원작 만화의 칼라 페이지
- 게임 OP 영상
- 주술 전투 장면들
```

### 2.3 피해야 할 스타일

❌ 밝고 귀여운 아동용 스타일
❌ 사실적인 사진 수준의 그래픽
❌ 과도하게 시니컬한 톤
❌ 게임의 세계관과 맞지 않는 스타일

## 3. 세력별 비주얼 아이덴티티 (Faction Visual Identity)

### 3.1 주술 고등학교 (Jujutsu High School)

```
특징:
• 색상: 파란색, 흰색, 은색
• 이미지: 교육, 희망, 정통성
• 복장: 교복 기반, 깔끔한 디자인
• 무기: 현대적, 정제된 주술

카드 프레임: 파란색 테두리
상징: 별, 원형 마크

예시 캐릭터:
- 유토 김: 파란 교복, 은색 머리
- 메이 젠인: 흰색 의복, 파란 악센트
```

### 3.2 주술협회 (Jujutsu Society)

```
특징:
• 색상: 검은색, 금색, 빨강
• 이미지: 권력, 경험, 엄격함
• 복장: 정장, 전통적 한복 요소
• 무기: 고전적, 세련된 주술

카드 프레임: 금색 테두리
상징: 삼각형, 고전 도장

예시 캐릭터:
- 겟토 스그루: 검은 정장, 보라 빛
- 나나미 켄토: 회색 정장, 차분한 느낌
```

### 3.3 저주 세력 (Curse Faction)

```
특징:
• 색상: 검은색, 진한 보라, 녹색
• 이미지: 혼돈, 위협, 이질적
• 복장: 기이한 의상, 비인간적 모습
• 무기: 이상한 주술

카드 프레임: 검은색 테두리 + 보라 빛
상징: 뒤틀린 모양, 불규칙한 선

예시 캐릭터:
- 류 준: 검은 갑옷 같은 의상
- 마히토: 파란 피부, 이상한 모양
```

## 4. 카드 프레임 내 구도 (Composition Guide)

### 4.1 기본 원칙

```
┌────────────────────┐
│                    │  주요 요소는 중앙 상단부
│       명확한        │  (카드명 영역 아래)에 배치
│     실루엣 분리     │
│                    │  배경은 명확한
│   배경과 구분      │  색상 대비로
│                    │  주인공과 분리
└────────────────────┘

카드 이미지 안전 영역:
┌─────────────────────────┐
│ ▒▒▒ 생략금지 영역       │
│ ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒  │
│ ▒▒ ┌────────────┐ ▒▒▒  │
│ ▒▒ │            │ ▒▒▒  │
│ ▒▒ │ 안전 영역  │ ▒▒▒  │
│ ▒▒ │            │ ▒▒▒  │
│ ▒▒ └────────────┘ ▒▒▒  │
│ ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒  │
│ ▒▒▒ 모서리 3mm는 컷오프│
└─────────────────────────┘

(Bleed 영역과 Safe Zone 참고)
```

### 4.2 카드 타입별 구도 지침

#### 저주술사 (Jujutsu User)

```
전신 또는 상반신 샷
• 캐릭터가 명백하게 드러나야 함
• 배경은 보조적 역할
• 포즈는 능력을 시사하는 동적 포즈
• 무기나 도구가 있으면 강조

예시 포즈:
  - 주술 사용하는 자세
  - 준비 자세
  - 능력 발동 중인 모습
```

#### 주술 (Cursed Technique)

```
효과를 표현하는 추상적 또는 액션 중심
• 카드의 효과를 시각화
• 과도한 그래픽 효과 가능
• 캐릭터가 있으면 보조 역할
• 에너지, 빛, 파괴의 표현

예시:
  - 바람이 휘날리는 효과
  - 저주가 퍼지는 모습
  - 폭발이나 파괴 장면
```

#### 저주물 (Cursed Object)

```
물체 중심, 세부 묘사
• 아이템/도구의 명확한 표현
• 텍스처와 질감 강조
• 배경은 최소한으로
• 신비한 빛 또는 에너지 입혀짐

예시:
  - 저주받은 검
  - 신비로운 보주
  - 기이한 인형
```

## 6. 타이포그래피 (Typography)

### 6.1 폰트 선택

```
영문:
• 제목: Gothic (굵고 현대적)
• 본문: Sans-serif (명확하고 읽기 쉬움)

한글:
• 제목: 고딕 또는 검은 글씨 (굵음)
• 본문: 아리얼 한글, 나눔고딕 (명확함)

금지사항:
❌ 손글씨 스타일
❌ 장식적 폰트
❌ 가독성 떨어지는 폰트
```

### 6.2 텍스트 배치

```
카드 이름: 상단, 16pt, 검은색 또는 금색
카드 타입: 우측 상단, 10pt, 회색
비용: 좌측 상단, 24pt, 금색
텍스트: 중앙 하단, 8pt, 검은색
```

## 8. 자산 추적 (Asset Tracking & Management)

### 8.1 파일 명명 규칙 (Naming Convention)

```
Format: SET##_###_CardName_Status_v##.ext

예시:
✓ SET01_005_YutoKim_Sketch_v01.psd
✓ SET01_005_YutoKim_Coloring_v02.psd
✓ SET01_005_YutoKim_Final_v03.psd
✓ SET01_005_YutoKim_Final_Print_v03.tiff

규칙:
• SET##: 세트 번호 (SET01, SET02 등)
• ###: 카드 번호 (001~200)
• CardName: 영문 카드명 (공백 없음)
• Status: Sketch / Coloring / Final / Print
• v##: 버전 번호 (v01, v02, v03...)
• ext: 파일 확장자 (psd, tiff, jpg)
```

### 8.2 자산 상태 관리 (Kanban Board)

```
Backlog → Assigned → Sketching → Revision → Coloring → Final → Done

각 상태별 체크리스트:
✓ Assigned: 아티스트 배정, 브리프 전달
✓ Sketching: 스케치 완성, 디자인 승인
✓ Coloring: 채색 완성, 색상 검수
✓ Final: 최종 완성, QA 검수
✓ Done: 파일 정리, 아카이브 저장
```

## 10. 아티스트 온보딩 가이드

### 10.1 신규 아티스트 체크리스트

```
□ 1. 이 아트 바이블 읽기 (필수)
□ 2. 무드 보드 이미지 검토
□ 3. 세력 가이드라인 숙지
□ 4. 카드 샘플 3~5개 검토
□ 5. 파일 명명 규칙 이해
□ 6. 피드백 프로세스 이해
□ 7. 첫 카드 스케치 승인 후 진행
```

### 10.2 일반적인 실수 사항

```
❌ 배경 과도하게 상세함 → 주체 돋보이지 않음
❌ 색상 팔레트 무시 → 세력 정체성 상실
❌ 포즈가 정적 → 동동성 부족
❌ 텍스트 영역 침범 → 카드 정보 가림
❌ 원작과 다른 스타일 → 팬의 불만
```

## 11. 참고 자료 및 라이센스

### 11.1 외부 참고 자료

- 주술회전 애니메이션 공식 이미지
- 주술회전 원작 만화 공식 일러스트
- 게임 OP/ED 영상

### 11.2 저작권 및 계약

- 모든 완성 일러스트는 "업무상 저작물(Work-for-Hire)"
- 계약서에 2차 저작물 작성권 포함 필요
- 아티스트 크레딧은 카드에 기재

## 12. 이 문서 사용 방법

### 12.1 역할별 읽기 순서

**신규 아티스트**
1. 1~3절 읽기 (목적 및 톤)
2. 해당 세력 가이드 정독 (3절)
3. 4절 구도 가이드 숙지
4. 카드 샘플 5개 검토

**아트 디렉터**
1. 전체 정독
2. 월 1회 가이드 업데이트 검토
3. 신규 아티스트 온보딩 관리

**게임 디자이너**
1. 1~3절만 읽기
2. 신규 카드 설계 시 세력 컨셉 확인
